Back office

Tools for curating the dataset and cleaning up tagging mistakes. · back to identifier

The flag identifier is the mapper-facing site. Everything below is maintenance work — curating attributes, fixing OSM tags that point at the wrong Wikidata entity, and filling Wikidata gaps so the dataset improves over time.

Curate this site's data

Edits made on any of these pages share one in-progress bag — classify colors on one page, fill in a description on another, and they export together as a single overrides.json.

  • Curate flag attributes → …

    One-flag-at-a-time queue editor for display name, flag:type, colors, iconography, shape, and description. Best for the "needs attention" workflow where you grind through whatever is missing.

  • Describe flags → …

    Browse every flag with its current description; type to edit or fill in. Filter by missing description, image presence, flag-entity classification, or filename heuristic; sort by OSM count or name.

  • Review colors → …

    Browse every flag with its current palette as chip toggles. Click chips to add or remove colors. Same filters and sort options as the describe page.

  • Review iconography → …

    Browse every flag with its current iconography tags as chip toggles (text, animal, cross, stars, stripes, etc.). Same filters and sort options as the describe page.

Fix OSM tagging

  • Review OSM-side fixes →

    QIDs tagged in OSM that point at the wrong Wikidata entity (subject instead of flag, redirected QIDs). Each row links to overpass-turbo for in-place retagging.

Fix Wikidata

  • Suggested Wikidata edits →

    Aggregated cleanup work: missing flag entities, missing P31 classification, missing P18 images, color mismatches between Wikidata and our curated palette.

  • Create flag entities →

    Multi-select review for subjects with a P41 flag image but no dedicated "flag of X" entity. Generates a QuickStatements batch to create them.
