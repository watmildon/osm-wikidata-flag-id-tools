Back office

Tools for curating the dataset and cleaning up tagging mistakes. · back to identifier

The flag identifier is the mapper-facing site. Everything below is maintenance work — curating attributes, fixing OSM tags that point at the wrong Wikidata entity, and filling Wikidata gaps so the dataset improves over time.

Curate this site's data

  • Curate flag attributes →

    Per-flag chip editor for display name, flag:type, colors, iconography, and shape. Saves edits to data/overrides.json.

  • Describe flags →

    Bulk review and edit of AI-written visual descriptions. Reads the current describe queue, exports diffs as describe-edits.json.

Fix OSM tagging

  • Review OSM-side fixes →

    QIDs tagged in OSM that point at the wrong Wikidata entity (subject instead of flag, redirected QIDs). Each row links to overpass-turbo for in-place retagging.

Fix Wikidata

  • Suggested Wikidata edits →

    Aggregated cleanup work: missing flag entities, missing P31 classification, missing P18 images, color mismatches between Wikidata and our curated palette.

  • Create flag entities →

    Multi-select review for subjects with a P41 flag image but no dedicated "flag of X" entity. Generates a QuickStatements batch to create them.
